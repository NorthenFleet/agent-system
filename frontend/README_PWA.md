# 团队看板前端 - PWA 与响应式优化

## 更新内容 (2026-04-01)

### 1. 响应式设计 - 7 个断点适配

已实现从 320px 到 2560px 的完整响应式覆盖：

| 断点 | 屏幕宽度 | 设备类型 | 布局特点 |
|------|----------|----------|----------|
| 1 | ≤320px | iPhone SE, 小屏手机 | 单列布局，隐藏侧边栏，精简内容 |
| 2 | 321-480px | 标准手机 | 双列统计卡片，隐藏侧边栏 |
| 3 | 481-768px | 平板竖屏 | 双列布局，横屏时显示侧边栏 |
| 4 | 769-1024px | 平板横屏/小笔记本 | 固定侧边栏，双列智能体 |
| 5 | 1025-1280px | 标准笔记本 | 三列统计卡片，双列布局 |
| 6 | 1281-1680px | 大笔记本/小台式机 | 四列统计卡片，三列智能体 |
| 7 | ≥1681px | 大台式机/4K 显示器 | 最大化布局，大字体 |

### 2. PWA 配置

#### manifest.json
- 应用名称：OpenClaw 团队信息看板
- 主题色：#58a6ff (GitHub 蓝)
- 背景色：#0d1117 (GitHub 暗色)
- 显示模式：standalone (独立应用)
- 快捷方式：仪表盘、智能体、设备

#### 图标
已生成 8 种尺寸的 PNG 图标 (72x72 - 512x512)
位置：`/icons/`

#### Service Worker (sw.js)
- **缓存策略**:
  - 静态资源：缓存优先 (Cache First)
  - API 请求：网络优先，失败时返回缓存 (Network First)
  - WebSocket：直接通过网络
- **离线支持**: 核心资源预缓存
- **自动更新**: 检测新版本并提示

### 3. 移动端手势支持

| 手势 | 功能 |
|------|------|
| 右滑 | 打开侧边栏 (移动端) |
| 左滑 | 关闭侧边栏 |
| 上滑 | 预留刷新功能 |
| 下滑 | 预留通知功能 |
| 双指放大 | 预留放大视图 |
| 双指缩小 | 预留缩小视图 |
| 双击 | 防止缩放 |

### 4. 移动端优化

- **触摸目标**: 最小 44x44px (符合 WCAG 标准)
- **安全区域**: 适配 iPhone X+ 的刘海屏和底部横条
- **触摸反馈**: 点击缩放效果
- **菜单按钮**: 移动端固定定位的汉堡菜单
- **遮罩层**: 侧边栏打开时的背景遮罩

- Font Awesome 6

### 8. 部署说明

1. 确保服务器支持 HTTPS (PWA 要求)
2. 配置正确的 MIME 类型:
   - `.json` → `application/json`
   - `.js` → `application/javascript`
3. 设置缓存头:
   - `index.html`: `no-cache`
   - `sw.js`: `no-cache`
   - 静态资源：长期缓存
