# PRD: 前端任务看板升级 - 全自动任务开发流程

**版本**: 1.0  
**创建日期**: 2026-04-15  
**负责人**: 孙总  
**优先级**: P0

---

## 1. 项目概述

### 1.1 背景
当前任务看板缺少实时任务状态展示和人工确认环节。需要升级前端界面，支持：
- 实时显示任务开发进度
- 每轮方案制定后等待孙总确认
- 全自动任务开发流程可视化

### 1.2 目标
- 实现任务状态实时展示
- 添加方案确认交互
- 支持全自动任务开发流程

### 1.3 范围
**包含**:
- 任务列表实时刷新
- 方案确认对话框
- 任务进度可视化
- 自动/手动模式切换

**不包含**:
- 后端 API 改造 (复用现有接口)
- 移动端适配

## 2. 功能需求

### 2.1 功能列表

| 编号 | 功能名称 | 描述 | 优先级 | 预计工时 |
|------|----------|------|--------|----------|
| F001 | 任务列表实时刷新 | 自动刷新任务状态 | P0 | 4h |
| F002 | 方案确认对话框 | 显示方案并等待确认 | P0 | 4h |
| F003 | 任务进度可视化 | 甘特图/进度条展示 | P1 | 6h |
| F004 | 自动/手动模式切换 | 切换开发模式 | P1 | 2h |

### 2.2 功能详情

#### F001: 任务列表实时刷新

**用户故事**:
> 作为孙总，我希望实时看到任务状态更新，以便了解开发进度。

**验收标准**:
- [ ] 每 30 秒自动刷新任务列表
- [ ] 任务状态变更时高亮显示
- [ ] 支持手动刷新按钮

**技术要点**:
- 技术栈：Vue 3 + WebSocket/轮询
- 接口：GET /api/v1/tasks
- 数据：任务状态、进度、负责人

#### F002: 方案确认对话框

**用户故事**:
> 作为孙总，我希望在每轮方案制定后确认是否继续，以便控制开发方向。

**验收标准**:
- [ ] 方案生成后弹出确认对话框
- [ ] 显示方案详情 (Task Spec + Dev Spec)
- [ ] 支持"继续执行"和"修改方案"操作

**技术要点**:
- 技术栈：Element Plus Dialog
- 接口：GET /api/v1/specs/{id}
- 数据：方案内容、创建时间、负责人

#### F003: 任务进度可视化

**用户故事**:
> 作为孙总，我希望直观看到任务进度，以便评估完成时间。

**验收标准**:
- [ ] 显示任务进度条
- [ ] 支持甘特图视图
- [ ] 显示里程碑标记

**技术要点**:
- 技术栈：ECharts / dhtmlxGantt
- 接口：GET /api/v1/tasks/progress
- 数据：任务开始/结束时间、进度百分比

#### F004: 自动/手动模式切换

**用户故事**:
> 作为孙总，我希望切换自动/手动模式，以便灵活控制开发流程。

**验收标准**:
- [ ] 模式切换开关
- [ ] 自动模式下任务自动执行
- [ ] 手动模式下每步需确认

**技术要点**:
- 技术栈：Vue 3 + Pinia 状态管理
- 接口：PUT /api/v1/mode
- 数据：当前模式、配置

## 4. 技术架构

### 4.1 系统架构图
```
前端 (Vue 3 + Element Plus)
    ↓ WebSocket/轮询
后端 API (FastAPI)
    ↓
数据库 (SQLite)
```

### 4.2 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Vue 3 + Element Plus | UI 框架 |
| 状态管理 | Pinia | 全局状态 |
| 图表 | ECharts | 可视化 |
| 后端 | FastAPI | API 服务 |

### 4.3 数据模型

```sql
-- 任务表
CREATE TABLE tasks (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    status TEXT, -- pending/in_progress/completed/waiting_approval
    progress REAL,
    assignee TEXT,
    created_at DATETIME,
    updated_at DATETIME
);

-- 方案表
CREATE TABLE specs (
    id TEXT PRIMARY KEY,
    task_id TEXT,
    type TEXT, -- task_spec/dev_spec
    content TEXT,
    status TEXT, -- pending/approved/rejected
    created_at DATETIME
);
```
