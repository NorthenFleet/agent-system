#### 核心功能

1. **空闲检测机制**
   - 自动识别 `status=idle` 或 `current_task=待分配` 的智能体
   - 记录空闲开始时间
   - 计算并格式化空闲时长 (秒/分钟/小时/天)

2. **智能任务匹配**
   - 基于职责 (responsibilities) 关键词匹配
   - 基于角色 (后端/前端/架构/测试) 特殊匹配
   - 自动推断任务类型 (从标题/描述)
   - 评分系统 (最高 40 分)

3. **开发计划生成**
   - 个性化下一步行动 (2-3 个具体步骤)
   - 时间估算 (基于任务类型)
   - 优先级计算 (基于空闲时长)

## 📊 技术实现

### 架构设计

```
┌─────────────────┐     ┌──────────────────┐
│   前端 Vue 3    │────▶│  FastAPI 后端    │
│  Element Plus   │◀────│                  │
└─────────────────┘     └────────┬─────────┘
                                 │
                    ┌────────────┼────────────┐
                    │            │            │
           ┌────────▼────┐ ┌────▼─────┐ ┌────▼──────┐
           │idle_agent   │ │  data    │ │  task     │
           │_manager.py  │ │_manager  │ │_queue     │
           └─────────────┘ └──────────┘ └───────────┘
```

### 数据流

1. **空闲检测**: 定时扫描 → 记录空闲时间 → 计算时长
2. **任务匹配**: 获取任务 → 职责匹配 → 角色加成 → 评分排序
3. **计划生成**: 选择最佳任务 → 生成步骤 → 估算时间 → 计算优先级
4. **任务分配**: 用户点击 → API 调用 → 更新状态 → 记录历史

### 数据持久化

- **状态文件**: `backend/data/idle_agent_state.json`
  - `agent_idle_since`: 智能体空闲开始时间
  - `last_scan`: 最后扫描时间
  - `assignment_history`: 分配历史记录 (最近 100 条)

## 🚀 使用指南

### 查看开发计划

1. 打开看板：http://localhost:3020
2. 点击侧边栏 **📋 开发计划**
3. 查看空闲智能体和推荐计划

### 分配任务

1. 在开发计划页面找到推荐的智能体 → 任务对
2. 点击 **分配任务** 按钮
3. 系统自动:
   - 更新智能体状态为 `busy`
   - 设置 `current_task` 为任务标题
   - 记录分配历史

### 定时检查

在 `HEARTBEAT.md` 中添加:

```markdown
## 开发计划检查
- 每 4 小时检查空闲智能体
- API: `curl http://localhost:3020/api/idle-agents/stats`
```
